Frontend SPOF

This site serves as a demo of different Frontend SPOF issues and how to fix them. The video below provides a quick example of what users may experience when a site has a Frontend SPOF.

What is Frontend SPOF?

The earliest reference to the term Frontend SPOF I can find is by Steve Sounders in 2010, in his blog post about the topic. This topic deals optiming webpage rendering in at the browser level and how assets are requested and rendered. The simplest form of the issue can be described as:

Requiring a third party asset which that render blocking.
This third party asset can be anything, such as CSS, JavaScript, or Fonts, so long as the browser considers that asset part of the critical rendering path.
If for any reason the browser cannot load a critical asset, to the user, your site is effectively down.
The problem with critical third party assets is of course, you have no control over them. The third party could decide to remove the asset, the site could be overloaded or down, or something could be blocking access to the third party server. For instance, it is not uncommon for a corporate firewall to block access to social networks such as Facebook and Twitter. If your site includes social widgets in a critical manner, the site may not work users who have restricted access to the web.

Testing for SPOF

There are a few different methods of testing for Frontend SPOF.

  • www.webpagetest.org provides the Google Chrome extension SPOF-O-Matic. This extension adds a small button in Chrome's toolbar which will show how many likely third-pary SPOF assets are on a page. It also provides a method of simulating the resources being unavailable.
  • www.webpagetest.org also provides a method of testing Frontend SPOF on it's site. When starting a new test, the SPOF tab can be used to provide domains which should be unavaiable.
  • blackhole.webpagetest.org can also be used point domains to an unreachable server. This is useful for local testing if you can edit your hosts file.
  • SPOFCheck is a command line tool built in Node.js which detects possible Frontend SPOF assets.
