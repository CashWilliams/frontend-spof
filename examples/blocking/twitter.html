Twitter Widget SPOF

This page includes a typical twitter share widget, in this case a Tweet Button, included in a manner that causes the browser to wait for the JavaScript provided by Twitter.com.

Note: This method of including the Tween Button is done incorrectly on purpose. Twitter's documentation describes how to correctly load the JavaScript asynchronously.

The Button!

Tweet

The Code

There are two parts to providing a Twitter Button, creating an <a> tag and including the reqired JavaScript. The <a> tag is standard, with a specific CSS class and optional data-size attributes.

<a class="twitter-share-button" href="https://twitter.com/intent/tweet?text=Hello%20world" data-size="large">Tweet</a>

Incuding the JavaScript is where the problem comes in. The standard method of including external JavaScript is to simply place a >script< tag in the >head< section of the HTML. This is the method the widgets.js file is included on this page:

<script src="https://platform.twitter.com/widgets.js"></script>

Using this method, the browser has no idication if this JavaScript is fundamental to rendering the page. The browser has to assume the page cannot function without this JavaScript, which would be the case if this file were something like jQuery or AngularJS.
