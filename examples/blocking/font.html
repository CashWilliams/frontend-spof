Web Font SPOF

The original version of this site was made with GitHub Pages using the Architect theme. On most pages, the custom Google Web Font Architects Daughter was removed. However this page still contains the CSS link which loads it. Loading the font in this way causes the browser to wait for the CSS file to be loaded from https://fonts.google.com.

The Font!

Assuming your browser properly loaded the font, the page should have headers that appear handwritten like the image below.

The Code

Google Fonts' documentation offer three methods of including a font. The 'Standard' method is used in this case, which is to include a <link> tag in the <head> section of the HTML.

<link href='https://fonts.googleapis.com/css?family=Architects+Daughter' rel='stylesheet' type='text/css'>

Using this method, the browser has assumes the CSS (or font) is needed to rendering the page, which is probably a valid assumption from the designer's perspective. However, if the choice is between seeing the without the custom font or not seeing the site at all, it stands to reason the font is not essential.
