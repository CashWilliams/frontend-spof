Web Font SPOF

Google's documentation on their Web Font site contains three methods of including a font. The third method is a JavaScript only them that loads the font file after the page has rendered. This method does come with the downside that the browser will display the text without the font, and then re-render the text once the font file has downloaded.

The Code

The basic idea is to load the font asynchronously. There are a few methods of achiving this. Google Fonts' documentation offers a JavaScript method of including their fonts, which is how this page is constructed. Using this method, the browser must have JavaScript enabled. The following code can be found in the <head> tag of this page: 

<script type="text/javascript">
  WebFontConfig = {
    google: { families: [ 'Architects+Daughter::latin' ] }
  };
  (function() {
    var wf = document.createElement('script');
    wf.src = 'https://ajax.googleapis.com/ajax/libs/webfont/1/webfont.js';
    wf.type = 'text/javascript';
    wf.async = 'true';
    var s = document.getElementsByTagName('script')[0];
    s.parentNode.insertBefore(wf, s);
})();
</script>

An alternative method is to use the preload value in the rel attribute of the <link> tag. However, not all browsers support this method yet, which is why the polyfill loadCSS was built. The loadCSS site contains documentation for installation and methods of use.

Note: The recommended SPOF-O-Matic Chrome extenstion will still show any CSS files using this alternative method as SPOF, they are not however.
